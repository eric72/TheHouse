Guide développeur — TheHouse

Documentation d’onboarding pour reprendre le code C++ et les systèmes du projet. Moteur : Unreal Engine 5.7 · Module : TheHouse.

Vision du jeu

Gestion de casino type tycoon / RTS (caméra aérienne, placement). FPS en activité secondaire (exploration, missions) — le FPS ne gère pas la mécanique de gestion du casino. Voir aussi Docs/PROJECT_OVERVIEW.md et Docs/SYSTEMS.md.

Le C++ couvre surtout : caméra RTS, bascule FPS, placement d’objets, sélection, slots PNJ sur les objets, occlusion caméra, murs.

Architecture globale

ATheHouseGameModeBase
  DefaultPawn = ATheHouseCameraPawn
  PlayerController = ATheHousePlayerController
  HUD = ATheHouseHUD

UTheHouseGameViewportClient → molette → zoom RTS (avant PlayerInput)
ATheHousePlayerController → possède RTS ou FPS, placement, sélection

Configuration & point d’entrée

  • TheHouse.uprojet : Enhanced Input désactivé — mappings classiques dans DefaultInput.ini.
  • DefaultEngine.ini : GameViewportClientClassName=/Script/TheHouse.TheHouseGameViewportClient (molette).
  • GameMode carte : souvent BP_HouseGameMode — le PlayerController doit rester compatible avec ATheHousePlayerController.

Entrées (touches, souris)

Définitions dans Config/DefaultInput.ini ; le PC force un UInputComponent classique pour les BindAction / BindAxis. Détail complet : GUIDE_FR.md (section Entrées : touches et souris).

Placement preview (RTS) : molette — zoom ou pivotement ?

Uniquement si un preview de placement est actif (EPlacementState::Previewing) et que tu possèdes le pion RTS.

Le preview, c’est l’acteur ATheHouseObject instancié pour la prévisualisation (PreviewActor sur le PlayerController) : le même type d’objet que celui qu’on placera, avant validation.

TouchesEffet
Molette sans ShiftZoom / dézoom caméra RTS (longueur du spring arm) : axe Zoom + TheHouse_ApplyWheelZoom — comme hors placement.
Shift (gauche ou droite) + molettePas de zoom : TheHouse_ApplyWheelZoom sort tout de suite. La rotation s’applique à ce ATheHouseObject de preview : PlayerTick détecte les crans molette haut/bas et fait tourner cet acteur sur l’axe yaw par pas de PlacementRotationStepDegrees (défaut 15° sur le PC).

Précision : ce n’est pas un zoom « sur le pivot » — c’est une rotation (orientation) de ce fantôme ATheHouseObject au sol. Le pas est réglable sur ATheHousePlayerController (PlacementRotationStepDegrees).

Autres repli code : confirmation placement au clic gauche dans PlayerTick ; timer ~120 Hz qui poll ZQSD si besoin.

GameMode (ATheHouseGameModeBase)

Pawn par défaut = caméra RTS, PC = ATheHousePlayerController, HUD = ATheHouseHUD. Dans InitGame, si le GameMode Blueprint laisse le PC Epic par défaut, le code corrige et log une erreur — sans cela les entrées semblent mortes.

Viewport client

UTheHouseGameViewportClient::InputAxis traite la molette avant les soucis Game+UI. Appelle TheHouse_ApplyWheelZoom et retourne true sans Super pour ce cas afin d’éviter le double zoom.

PlayerController

Rôles

  • Références RTSPawnReference et ActiveFPSCharacter.
  • RTS : déplacement plan, zoom (longueur du spring arm), yaw sur le contrôleur, pitch sur le boom du ATheHouseCameraPawn.
  • Timer TheHouse_PollInputFrame (~120 Hz) : polling clavier / viewport si PlayerInput ou focus Slate bloquent ZQSD.
  • Placement : preview sous le curseur, snap grille, rotation Shift+molette, confirmation clic.
  • Sélection : rectangle HUD ; acteurs ITheHouseSelectable dans la zone → OnSelect.

RTS ↔ FPS

  • SwitchToRTS : annule placement, aligne la caméra RTS sur le FPS (si hauteur OK), repossède le RTS, détruit le FPS, curseur visible.
  • SwitchToFPS : spawn du personnage, repossession, masque certains cubes de test, mode jeu seul.
  • DebugSwitchToFPS : trace depuis la souris pour trouver le sol.

Zoom

Paramètres : RTSWheelZoomMultiplier, RTSZoomSpeed, RTSMinArmDeltaPerWheelEvent, interpolation, zoom vers curseur, etc. En placement : Shift+molette = rotation du preview (pas de zoom).

Pion RTS (ATheHouseCameraPawn)

Spring arm + caméra + UFloatingPawnMovement. Le pitch RTS est sur RTSArmPitchDegrees (boom), pas uniquement sur la control rotation. Occlusion : fade matériau entre caméra et pawn (CheckCameraOcclusion).

Personnage FPS

ATheHouseFPSCharacter : ACharacter + ITheHouseSelectable. bCanBeSelected pour désactiver la sélection RTS.

HUD

ATheHouseHUD dessine le rectangle. La résolution des acteurs sélectionnés est dans le PlayerController.

ATheHouseObject

  • Zone d’exclusion : PlacementExclusionHalfExtent, PlacementExclusionCenterOffset — empêche le chevauchement avec d’autres ATheHouseObject.
  • Visualizer : mesh de prévisualisation (souvent cube moteur), sans collision.
  • États : EObjectPlacementState (Valid, OverlapsObject, OverlapsWorld, OverlapsGrid).
  • PNJ : FTheHouseNPCInteractionSlot (socket + PurposeTag), réservation / libération de slots.
  • Sélection : surbrillance (Custom Depth / stencil).

Placement & grille

Trace sous le curseur, normale du sol ≥ MinUpNormalZ, snap XY optionnel. EvaluatePlacementAt teste objets + monde. Cache OccupiedGridCells pour les cellules occupées. ConfirmPlacement finalise et marque les cellules.

Console : TheHouse.Placement.Debug = logs de flux.

ITheHouseSelectable

OnSelect, OnDeselect, IsSelectable. Utilisé par objets, FPS, et acteurs Blueprint compatibles.

IA — UTheHouseObjectSlotUserComponent

États : Idle, MovingToSlot, Aligned, PlayingMontage. UseObjectSlot réserve un slot par tag, MoveTo via AIController, alignement interpolé, montage optionnel.

Debug

VariableEffet
TheHouse.Placement.DebugLogs placement (0/1).
TheHouse.FPS.DebugLogs trace / meshes devant la caméra FPS.

Pièges fréquents

  1. GameMode BP sans le bon PlayerController (corrigé en C++ mais à aligner).
  2. Enhanced Input réactivé dans DefaultInput.ini alors que le plugin est off.
  3. Event Tick Blueprint sans appeler Parent sur le PC / Pawn.
  4. Double chemin molette en plus du GameViewportClient.
  5. Surfaces trop pentues ou pas de hit sous le curseur → placement invalide.
