Guide développeur — TheHouse

Documentation d’onboarding pour reprendre le code C++ et les systèmes du projet. Moteur : Unreal Engine 5.7 · Module : TheHouse.

Vision du jeu

Gestion de casino type tycoon / RTS (caméra aérienne, placement). FPS en activité secondaire (exploration, missions) — le FPS ne gère pas la mécanique de gestion du casino. Voir aussi Docs/PROJECT_OVERVIEW.md et Docs/SYSTEMS.md.

Le C++ couvre surtout : caméra RTS, bascule FPS, placement d’objets, sélection, UI RTS (panneau, menus, ordres), slots PNJ sur les objets, socle PNJ (archetypes, registre, volume d’éjection), localisation (sous-système + scripts), occlusion caméra, murs.

Architecture globale

ATheHouseGameModeBase
  DefaultPawn = ATheHouseCameraPawn
  PlayerController = ATheHousePlayerController
  HUD = ATheHouseHUD

UTheHouseGameViewportClient → molette → zoom RTS (avant PlayerInput)
ATheHousePlayerController → possède RTS ou FPS, placement, sélection
ATheHouseNPCCharacter + ATheHouseNPCAIController → PNJ (pas le PlayerController)

Configuration & point d’entrée

  • TheHouse.uproject : Enhanced Input désactivé — mappings classiques dans DefaultInput.ini.
  • DefaultEngine.ini : GameViewportClientClassName=/Script/TheHouse.TheHouseGameViewportClient (molette).
  • GameMode carte : souvent BP_HouseGameMode — le PlayerController doit rester compatible avec ATheHousePlayerController.

Placement preview (RTS) : molette — zoom ou pivotement ?

Uniquement si un preview de placement est actif (EPlacementState::Previewing) et que tu possèdes le pion RTS.

Le preview, c’est l’acteur ATheHouseObject instancié pour la prévisualisation (PreviewActor sur le PlayerController) : le même type d’objet que celui qu’on placera, avant validation.

TouchesEffet
Molette sans ShiftZoom / dézoom caméra RTS (longueur du spring arm) : axe Zoom + TheHouse_ApplyWheelZoom — comme hors placement.
Shift (gauche ou droite) + molettePas de zoom : TheHouse_ApplyWheelZoom sort tout de suite. La rotation s’applique à ce ATheHouseObject de preview : PlayerTick détecte les crans molette haut/bas et fait tourner cet acteur sur l’axe yaw par pas de PlacementRotationStepDegrees (défaut 15° sur le PC).

Précision : ce n’est pas un zoom « sur le pivot » — c’est une rotation (orientation) de ce fantôme ATheHouseObject au sol. Le pas est réglable sur ATheHousePlayerController (PlacementRotationStepDegrees).

Autres repli code : confirmation placement au clic gauche dans PlayerTick ; timer ~120 Hz qui poll ZQSD si besoin.

GameMode (ATheHouseGameModeBase)

Pawn par défaut = caméra RTS, PC = ATheHousePlayerController, HUD = ATheHouseHUD. Dans InitGame, si le GameMode Blueprint laisse le PC Epic par défaut, le code corrige et log une erreur — sans cela les entrées semblent mortes.

Viewport client

UTheHouseGameViewportClient::InputAxis traite la molette avant les soucis Game+UI. Appelle TheHouse_ApplyWheelZoom et retourne true sans Super pour ce cas afin d’éviter le double zoom.

PlayerController

Rôles

  • Références RTSPawnReference et ActiveFPSCharacter.
  • RTS : déplacement plan, zoom (longueur du spring arm), yaw sur le contrôleur, pitch sur le boom du ATheHouseCameraPawn.
  • Timer TheHouse_PollInputFrame (~120 Hz) : polling clavier / viewport si PlayerInput ou focus Slate bloquent ZQSD.
  • Placement : preview sous le curseur, snap grille, rotation Shift+molette, confirmation clic.
  • Sélection : rectangle HUD ; acteurs ITheHouseSelectable dans la zone → OnSelect.

Sélection / panneau paramètres

  • Input_SelectPressed / Input_SelectReleased gèrent le clic gauche RTS ; Input_SelectDoubleClick bind double clic gauche pour rouvrir le panneau même si le 2e clic est absorbé par l’UMG.
  • TheHouse_IsCursorOverBlockingRTSViewportUI filtre l’UMG qui doit vraiment bloquer le monde. Les conteneurs plein écran purement décoratifs doivent rester en SelfHitTestInvisible / HitTestInvisible, pas en Visible.
  • Blueprint à renseigner dans BP_HouseController : PlacedObjectSettingsWidgetClass, RTS Main Widget Class, RTS Context Menu Widget Class.

Menus contextuels PNJ / ordres

  • Clic gauche : sélection (PNJ / objets ITheHouseSelectable).
  • Clic droit : ouvre un menu selon le contexte (objet, PNJ “inspect”, ou menu ordres si au moins un PNJ est sélectionné).
  • Propriétés BP (PlayerController) : NPCRTSContextMenuWidgetClass (menu PNJ), NPCOrderContextMenuWidgetClass (menu ordres).
  • Actions dynamiques : surcharger GatherNPCOrderActionsOnGround / ExecuteNPCOrderOnGroundAction (ou équivalents objet/PNJ) dans le BP du PlayerController.

RTS ↔ FPS

  • SwitchToRTS : annule placement, aligne la caméra RTS sur le FPS (si hauteur OK), repossède le RTS, détruit le FPS, curseur visible.
  • SwitchToFPS : spawn du personnage, repossession, masque certains cubes de test, mode jeu seul.
  • DebugSwitchToFPS : trace depuis la souris pour trouver le sol.

Zoom

Paramètres : RTSWheelZoomMultiplier, RTSZoomSpeed, RTSMinArmDeltaPerWheelEvent, interpolation, zoom vers curseur, etc. En placement : Shift+molette = rotation du preview (pas de zoom).

Pion RTS (ATheHouseCameraPawn)

Spring arm + caméra + UFloatingPawnMovement. Le pitch RTS est sur RTSArmPitchDegrees (boom), pas uniquement sur la control rotation. Occlusion : fade matériau entre caméra et pawn (CheckCameraOcclusion).

Personnage FPS

ATheHouseFPSCharacter : ACharacter + ITheHouseSelectable. bCanBeSelected pour désactiver la sélection RTS.

  • Contrôles FPS (remappables) : TheHouseFPSJump, TheHouseFPSRun, TheHouseFPS_Crouch, TheHouseFPSFire.
  • Caméra sur tête : socket (ex. HeadCameraEmplacement) + anti-clipping (cache tête/nuque pour le joueur local).
  • Animation locomotion : UTheHouseLocomotionAnimInstance (Speed/Direction, états air/landing, AimOffset yaw/pitch).
  • PV & mort : UTheHouseHealthComponent (ApplyDamage), ragdoll auto.
  • Combat : ATheHouseWeapon + ATheHouseProjectile (muzzle socket Muzzle), fallback hitscan.

HUD

ATheHouseHUD dessine le rectangle. La résolution des acteurs sélectionnés est dans le PlayerController.

ATheHouseObject

  • Zone d’exclusion : PlacementExclusionHalfExtent, PlacementExclusionCenterOffset — empêche le chevauchement avec d’autres ATheHouseObject.
  • Visualizer : mesh de prévisualisation (souvent cube moteur), sans collision.
  • États : EObjectPlacementState (Valid, OverlapsObject, OverlapsWorld, OverlapsGrid).
  • PNJ : FTheHouseNPCInteractionSlot (socket + PurposeTag), réservation / libération de slots.
  • Sélection : surbrillance (Custom Depth / stencil).
  • Panneau paramètres : cocher bOpenParametersPanelOnLeftClick sur le BP de l’objet pour autoriser l’ouverture au clic / double clic RTS.
  • Économie objet : PurchasePrice = prix d’achat catalogue. SellValue = prix de revente si renseigné ; si SellValue == 0, la vente reprend PurchasePrice.

Placement & grille

Trace sous le curseur, normale du sol ≥ MinUpNormalZ, snap XY optionnel. EvaluatePlacementAt teste objets + monde. Cache OccupiedGridCells pour les cellules occupées. ConfirmPlacement finalise et marque les cellules.

Console : TheHouse.Placement.Debug = logs de flux.

ITheHouseSelectable

OnSelect, OnDeselect, IsSelectable. Utilisé par objets, FPS, et acteurs Blueprint compatibles.

IA — UTheHouseObjectSlotUserComponent

États : Idle, MovingToSlot, Aligned, PlayingMontage. UseObjectSlot réserve un slot par tag, MoveTo via AIController, alignement interpolé, montage optionnel.

Système PNJ (architecture)

Référence complète : GUIDE_FR.md → Système PNJ.

Les PNJ du casino ne sont pas pilotés par le ATheHousePlayerController. Ils utilisent ATheHouseNPCAIController et ATheHouseNPCCharacter. Les données par métier vivent sous Source/TheHouse/NPC/Archetypes/ : classe racine UTheHouseNPCArchetype (non abstraite pour le picker d’assets) et trois types d’assets concrets (Staff, Customer, Special). Le salaire staff est en StaffMonthlySalary sur le personnage (modifiable UI) ; initialisé depuis DefaultMonthlySalary sur l’archetype Staff. UTheHouseNPCSubsystem : GetNPCsByCategory. ITheHouseNPCIdentity pour interroger catégorie / archetype.

Pour utiliser tables et machines : ajouter le composant UTheHouseObjectSlotUserComponent au Blueprint PNJ et appeler UseObjectSlot avec le même PurposeTag que les slots définis sur les ATheHouseObject.

Compilation : si Live Coding bloque la build, fermer l’éditeur ou désactiver Live Coding avant de compiler le module C++.

Prochaines étapes (éditeur)

  1. Compiler le module TheHouse (sans Live Coding bloquant).
  2. Créer des Data Assets du bon sous-type (Staff / Customer / Special) dans NPC/Archetypes.
  3. Blueprint parent ATheHouseNPCCharacter : mesh, anim, NPC Archetype.
  4. Ajouter TheHouse Object Slot User sur ce Blueprint pour les objets casino.
  5. Vérifier le contrôleur IA (ATheHouseNPCAIController ou BP dérivé).
  6. Brancher Behavior Tree + Blackboard sur le contrôleur IA selon les besoins.
  7. Tester GetNPCsByCategory avec plusieurs PNJ.
  8. Raccorder salaires et flux argent à l’économie globale quand elle sera prête.

Debug

VariableEffet

Pièges fréquents

  1. GameMode BP sans le bon PlayerController (corrigé en C++ mais à aligner).
  2. Enhanced Input réactivé dans DefaultInput.ini alors que le plugin est off.
  3. Event Tick Blueprint sans appeler Parent sur le PC / Pawn.
  4. Double chemin molette en plus du GameViewportClient.
  5. Surfaces trop pentues ou pas de hit sous le curseur → placement invalide.
  6. Le panneau paramètres “ne s’ouvre pas” : vérifier PlacedObjectSettingsWidgetClass sur le PlayerController BP, la collision Visibility de l’objet, puis bOpenParametersPanelOnLeftClick sur le BP de l’objet.
  7. Cycle jour/nuit : si InGameTimeSeconds démarre à 0, alors DayProgress01=0 (minuit) et le soleil peut être coupé (scène noire sans SkyLight / MoonLight). Voir Docs/Features/RTS_UI/README.md → Cycle jour/nuit.

Localisation (pipeline complet)

Créer une fois la cible Game dans le tableau de bord (Config/Localization/Game_*.ini). Utiliser Scripts/RunLocalizationStep.ps1 avec -Step … pour lancer UnrealEditor-Cmd avec -LiveCoding=false. Workflows complets (PO, CSV dialogue, import WAV, Wwise) : GUIDE_FR.md → Localisation.

Commande maître (remplacer <Etape> par Gather, Compile, Export, Import, ExportDialogueScript, ImportDialogueScript, ImportDialogue, Reports, ou RepairGatherIni) :

powershell -NoProfile -ExecutionPolicy Bypass -File "Scripts\RunLocalizationStep.ps1" -Step <Etape>

À lancer depuis la racine du projet (dossier contenant TheHouse.uproject et Scripts/).

Raccourcis : GatherLocalization.ps1 / CompileLocalization.ps1. Texte : Gather → Compile. Voix Unreal : ExportDialogueScript → … → ImportDialogue (config Game_ImportDialogue.ini) → Compile. Wwise reste un pipeline audio à part.
