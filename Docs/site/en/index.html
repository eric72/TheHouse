Developer Guide — TheHouse

Onboarding documentation for the C++ gameplay systems. Engine: Unreal Engine 5.7 · Module: TheHouse.

Game vision

Casino management / RTS (top-down camera, placement). FPS as a secondary activity — FPS does not run casino management. See Docs/PROJECT_OVERVIEW.md and Docs/SYSTEMS.md.

C++ focus: RTS camera, FPS switch, placement, selection, RTS UMG (main panel, context menus, NPC orders), NPC slots on objects, NPC foundation (archetypes, category registry, eject volume), localization (subsystem + CLI), camera occlusion, walls.

High-level architecture

ATheHouseGameModeBase
  DefaultPawn = ATheHouseCameraPawn
  PlayerController = ATheHousePlayerController
  HUD = ATheHouseHUD

UTheHouseGameViewportClient → mouse wheel → RTS zoom (before PlayerInput issues)
ATheHousePlayerController → possesses RTS or FPS, placement, selection
ATheHouseNPCCharacter + ATheHouseNPCAIController → NPCs (not the PlayerController)

Placement preview (RTS): wheel — zoom or rotate?

Only while a placement preview is active (EPlacementState::Previewing) and you possess the RTS pawn.

The preview is the ATheHouseObject actor spawned for placement preview (PreviewActor on the Player Controller): the same object type you are about to place, before you confirm.

InputEffect
Wheel without ShiftZoom in/out on the RTS camera (spring arm length): Zoom axis + TheHouse_ApplyWheelZoom — same as outside placement.
Shift (left or right) + wheelNo zoom: TheHouse_ApplyWheelZoom returns early. Rotation applies to that preview ATheHouseObject: PlayerTick uses discrete scroll up/down steps to rotate that actor’s yaw by PlacementRotationStepDegrees per notch (default 15° on the PC).

Note: this is not an optical “zoom on the pivot” — it rotates that ATheHouseObject placement ghost on the ground. Tune step size on ATheHousePlayerController (PlacementRotationStepDegrees).

Other code fallbacks: placement confirm via LMB in PlayerTick; ~120 Hz poll for WASD/ZQSD when needed.

GameMode (ATheHouseGameModeBase)

Default pawn = RTS camera, PC = ATheHousePlayerController, HUD = ATheHouseHUD. InitGame forces the correct PC class if a Blueprint left Epic’s default — otherwise input appears broken.

Viewport client

UTheHouseGameViewportClient::InputAxis handles the wheel before Game+UI swallow it. Calls TheHouse_ApplyWheelZoom and returns true without Super to avoid double zoom.

PlayerController

Responsibilities

  • RTSPawnReference and ActiveFPSCharacter.
  • RTS: plane movement, zoom (spring arm length), yaw on controller, pitch on boom.
  • TheHouse_PollInputFrame (~120 Hz) for WASD when PlayerInput / Slate focus fail.
  • Placement: cursor preview, grid snap, Shift+wheel rotation, click confirm.
  • Selection: HUD rect; ITheHouseSelectable actors → OnSelect.

Selection / parameters panel

  • Input_SelectPressed / Input_SelectReleased handle normal RTS left click; Input_SelectDoubleClick binds a real LMB double-click so the panel can reopen even if the second click is swallowed by UMG.
  • TheHouse_IsCursorOverBlockingRTSViewportUI filters only UMG that should truly block world input. Fullscreen decorative containers should stay SelfHitTestInvisible / HitTestInvisible, not Visible.
  • Blueprint values to set in BP_HouseController: PlacedObjectSettingsWidgetClass, RTS Main Widget Class, RTS Context Menu Widget Class.

NPC context menus & orders

  • Left click selects (NPCs / ITheHouseSelectable objects).
  • Right click opens a menu depending on context (placed object, NPC “inspect”, or orders when at least one NPC is selected).
  • PC Blueprint properties: NPCRTSContextMenuWidgetClass, NPCOrderContextMenuWidgetClass (plus object context menu class).
  • Dynamic actions: override GatherNPCOrderActionsOnGround / ExecuteNPCOrderOnGroundAction (or object/NPC variants) on a PlayerController subclass — see GUIDE_EN.md → RTS UI.

RTS ↔ FPS

  • SwitchToRTS: cancel placement, sync RTS to FPS height, repossess RTS, destroy FPS, show cursor.
  • SwitchToFPS: spawn character, possess, hide test cubes, game-only input.
  • DebugSwitchToFPS: trace from mouse to find ground.

Zoom

Tunables: RTSWheelZoomMultiplier, RTSZoomSpeed, RTSMinArmDeltaPerWheelEvent, interpolation, cursor-relative zoom. During placement: Shift+wheel rotates preview (no zoom).

RTS pawn (ATheHouseCameraPawn)

Spring arm + camera + UFloatingPawnMovement. RTS pitch lives on RTSArmPitchDegrees (boom). Occlusion fade via CheckCameraOcclusion.

FPS character

ATheHouseFPSCharacter: ACharacter + ITheHouseSelectable. bCanBeSelected toggles RTS selection.

HUD

ATheHouseHUD draws the box. Actor picking for selection is in the PlayerController.

ATheHouseObject

  • Exclusion zone: PlacementExclusionHalfExtent, PlacementExclusionCenterOffset — blocks overlap with other ATheHouseObject.
  • Visualizer: preview mesh (often engine cube), no collision.
  • States: EObjectPlacementState (Valid, OverlapsObject, OverlapsWorld, OverlapsGrid).
  • NPCs: FTheHouseNPCInteractionSlot (socket + PurposeTag), reserve / release.
  • Selection: highlight (Custom Depth / stencil).
  • Parameters panel: enable bOpenParametersPanelOnLeftClick on the object BP to allow opening from RTS click / double-click.
  • Object economy: PurchasePrice = catalog buy price. SellValue = resale price if set; when SellValue == 0, selling falls back to PurchasePrice.

Placement & grid

Trace under cursor; floor normal Z ≥ MinUpNormalZ; optional XY snap. EvaluatePlacementAt checks objects + world. OccupiedGridCells tracks grid cells. ConfirmPlacement finalizes and marks cells.

Console: TheHouse.Placement.Debug for verbose logs.

ITheHouseSelectable

OnSelect, OnDeselect, IsSelectable. Used by objects, FPS, and compatible Blueprint actors.

AI — UTheHouseObjectSlotUserComponent

States: Idle, MovingToSlot, Aligned, PlayingMontage. UseObjectSlot reserves by tag, MoveTo via AIController, interpolated align, optional montage.

NPC system (architecture)

Full reference: GUIDE_EN.md → NPC system.

Casino NPCs are not driven by ATheHousePlayerController. They use ATheHouseNPCAIController and ATheHouseNPCCharacter. Per-role data lives under Source/TheHouse/NPC/Archetypes/: root UTheHouseNPCArchetype (non-abstract so the asset picker lists subclasses) plus Staff / Customer / Special assets. Staff pay uses StaffMonthlySalary on the pawn (UI-editable), seeded from DefaultMonthlySalary on the Staff archetype. UTheHouseNPCSubsystem — GetNPCsByCategory. ITheHouseNPCIdentity for queries.

For tables and machines: add UTheHouseObjectSlotUserComponent to the NPC Blueprint and call UseObjectSlot with the same PurposeTag as slots on ATheHouseObject.

Builds: close the editor or disable Live Coding if C++ compilation is blocked.

Common pitfalls

  1. Blueprint game mode with wrong Player Controller (C++ fixes it — align BP anyway).
  2. Enhanced Input re-enabled in DefaultInput.ini while the plugin is off.
  3. Blueprint Event Tick without Parent on PC / Pawn.
  4. Second mouse-wheel path alongside GameViewportClient → double zoom.
  5. Steep surfaces or no hit under cursor → invalid placement.
  6. The parameters panel “does not open”: verify PlacedObjectSettingsWidgetClass on the Player Controller BP, the object’s Visibility collision, then bOpenParametersPanelOnLeftClick on the object BP.

Localization (full pipeline)

Create the Game target once in the Localization Dashboard (Config/Localization/Game_*.ini). Use Scripts/RunLocalizationStep.ps1 with -Step … so UnrealEditor-Cmd runs -LiveCoding=false (avoids hangs when the editor is open). Full workflows (PO, dialogue CSV, import WAVs, Wwise note): GUIDE_EN.md → Localization.

Master command (replace <Step> with Gather, Compile, Export, Import, ExportDialogueScript, ImportDialogueScript, ImportDialogue, Reports, or RepairGatherIni):

powershell -NoProfile -ExecutionPolicy Bypass -File "Scripts\RunLocalizationStep.ps1" -Step <Step>

Run from the project root (folder that contains TheHouse.uproject and Scripts/).

Shortcuts: GatherLocalization.ps1 / CompileLocalization.ps1. Typical text loop: Gather → Compile. VO: ExportDialogueScript → … → ImportDialogue (configure Game_ImportDialogue.ini paths) → Compile. Wwise remains a separate audio pipeline.
